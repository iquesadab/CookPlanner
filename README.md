# CookPlanner

## Descripción del proyecto

Este proyecto es una plataforma web para compartir y descubrir recetas de cocina.
Los usuarios pueden buscar recetas por ingredientes, tipo de comida o nivel de dificultad.
También pueden guardar sus recetas favoritas, comentar, crear sus propias versiones y planificar menús semanales.

La idea es tener una comunidad activa que comparta ideas y que la plataforma ayude a organizar mejor las comidas según gustos, presupuesto y lo que hay en casa.

## Alcance del proyecto

- Crear una web interactiva con HTML, CSS y JavaScript.
- Tres tipos de usuario: chef, usuario regular y administrador.
- Posibilidad de buscar, guardar, modificar y calificar recetas.
- Planificador de comidas y generación de lista de compras.
- Ranking de recetas populares y sistema de seguidores.

## Tipos de commits

Usaré los siguientes tipos para organizar bien los cambios en el código:
- [init]: commit especial que indica el inicio del repositorio.
- [new]: se ha creado un método o recurso en el programa que no existía antes del commit.
- [improved]: se mejoró la forma en que se hacía un método o como se mostraba algo.
- [fixed]: para corregir errores.
- [style]: para cambios de estilo (CSS, formato, sin afectar el funcionamiento).
- [wip]: trabajo en progreso, para commits temporales que no deberían llegar a producción.
- [docs]: para cambios en documentación (comentarios, README, guías).
